数组概述

javascript 中，数组即对象，与 java,c 不同，javascript 中数组是弱类型，即其元素可以是不同类型的值。数组是值的有序集合，数组中每值叫做元素，每个元素都有对应的索引

数组方法

Array.prototype.join
Array.prototype.reverse

数组被反转后的结果重新赋值给原数组，即会改变原数组。

Array.prototype.sort

数组被排序后的结果重新赋值给原数组，即会改变原数组
这里需要注意的是 sort 进行数值比较，sort 进行数值比较时，可以传入一个回调函数，回调函数中 return a-b 按升序排序，return b-a，按降序排序。

Array.prototype.concat 数组合并

合并后返回一个新数组，不会改变原数组

Array.prototype.slice 获取数组片断

slice 原数组不会改变
slice 有两个参数，截取数组开始的下标，以及截取至哪个下标的前一位。
这里arr.slice(-3,1);//[] 说明一下，源码中会把两个参数相加求 size，size 为负数的时候不处理，所以会返回空数组
arr.slice(-3,-1); //[4, 5] 只简单粗暴的说明上一点也不对，这个案例，两参数和为 -4，并未返回空数组，从源码中再说明一下处理方式，当第二个参数为负数时，会进行该数与长度的求和，根据结果再去截取数组片断，这里会截取的是 从下标 -3 即 下标为（-3+6）开始，截取至 （-1+6），但不包括 下标为 5 的元素。
具体，参看Array.prototype.slice()/);

Array.prototype.splice

splice 源数组不会被改变

Array.prototype.forEach 数组遍历

ES5 新特性，支持 IE9 以上浏览器
没有返回值，通常的应用场景是为一些相同的元素，绑定事件处理器
demo 中可以看出，三个参数，分别是数组元素、下标、数组本身

Array.prototype.map 数组映射

ES5 新特性，支持 IE9 以上浏览器
返回映射处理后的数组，不会改变原数组

Array.prototype.filter 数组过滤

ES5 新特性，支持 IE9 以上浏览器
返回根据条件过滤后的数组，不会改变原数组
继续前面的案例，想要实现的是一个过滤数组中非数值的元素
return 中给出过滤条件

Array.prototype.every 数组判断

判断一个数组是否符合指定条件，与 some 不同，需要每个元素都符合指定条件。
返回 Boolean

#####Array.prototype.some 数组判断

判断一个数组是否符合指定条件，与 every 不同，只要有某个元素符合指定条件即可。
返回 Boolean

Array.prototype.reduce 数组进行聚合，得出最终的一个结果

与 sort 进行数字数组排序一样，回调中两个参数分别代码当前值，接着对当前值进行操作
从打印结果可知，先将数组前两个元素进行求和，结果返回，接着再拿结果与第三个元素字符串进行拼接，后面同理。得出结果 3abc5

Array.prototype.reduceRight 数组进行聚合，得出最终的一个结果

与 reduce 不同的是它是从右向左取数据，先取两个元素进行对比，取出最大的数与下一个元素对比，最终找出最大的数

Array.prototype.indexOf 数组检索

用于检索数组中的元素，与字符串的 indexOf 方法用法相同
arr.indexOf(4,-1); //-1
k = Math.max(n >= 0 ? n : len - Math.abs(n), 0); n 是开始位置的传参，这里是 -1，k 的结果为 5
while (k < len) {
if (k in O && O[k] === searchElement) {
return k;
}
k++;
} //这里从数组的第五个元素开始，循环遍历 5<6 次，发现没有找到要找的 4. 所以返回 -1
具体请看文档

Array.prototype.lastIndexOf 数组检索

与 indexOf 不同的是，它是从数组的最后一个元素开始检索
