严格模式

一种特殊的执行模式，修复了部分语言上的不足，提供更强的错误检查，增强了安全性
1、前面说到了严格模式下不允许用 with，严格模式下使用 with 报Uncaught SyntaxError: Strict mode code may not include a with statement（详见 严格模式demo1）
2、不允许对未声明的变量赋值Uncaught ReferenceError: x is not defined（详见 严格模式demo2）
3、arguments 变为参数的静态副本（详见 严格模式demo3）
4、严格模式下 delete 删除 configurable（不可配置的属性）’Uncaught TypeError: Cannot delete property ‘p’ of’报错（详见 严格模式demo4）
5、严格模式下禁止变量名重复chrome 开发者工具招待 demo 未看到报错，这点需要重新论证（详见 严格模式demo5）
6、禁止八进制字面量Uncaught SyntaxError: Octal literals are not allowed in strict mode.（详见 严格模式demo6）
7、eval,arguments 为关键字，不能作为变量、函数名，否则报错Uncaught SyntaxError: Unexpected eval or arguments in strict mode；eval为独立作用域
chrome 开发者工具招待 demo 未看到报错，这点需要重新论证
8、一般函数调用，this 指向 null,而不是全局作用域。
9、若使用 apply/call，当传入 null 或 undefined时，this 指向 null 或 undefined，而不是全局对象
10、试图修改不可写属性（writable=false),在不可扩展的对象上添加属性时报错
11、arguments.caller,arguments.callee 禁用
