this 与函数

函数中的 this 与 调用它的环境相关，全局作用域中调函数，this 指向 window，而将 函数绑定到某一对象时，通过对象调用时， this 指向该对象。

this 与对象原型链

oN 的原型是 o，o 的方法 fn，返回 o.p1 与 o.p2 (this 这时是指向 o 的)，拼接（和）, oN，以 o 为原型的新对象，即 原型指向 o，对象 oN 调原型方法 fn 时，this 指向 oN，所以此时是 this.p1 为 oN.p1，以此类推。

抽像概念 - 有关浏览器实现标准

ECMA - 262 第三版标准规范定义

执行上下文（Execution Context EC);

类似栈的结构
执行顺序：EC0(funcEC1())->EC1(funcEC2())->EC2(funcEC3())
执行结果：EC0 EC1 EC2 EC3
随着函数调用，创建新的执行上下文，EC0 EC1 EC2 EC3 分别在不同的执行上下文中

变量对象（Variable Object VO）

VO 用于存储执行上下文中的：变量、函数声明、函数参数

由以下 demo 可知，变量及函数声明代码会产生两个作用域环境
函数声明是在 globalContext 中的，所以会被前置。

激活对象存在于函数作用域中

解析以下 demo
首先会创建两个执行环境，即 VO、AO
代码初始化阶段，AO 执行顺序：

  1. 函数参数（未传入，初始化值为 undefined）;
  2. 函数声明（命名冲突会发生覆盖）；
  3. 变量声明（初始化值为 undefined 时，若发生命名冲突，忽略）。
    函数表达式不会记入 AO 环境，所以外部无法执行通过表达式定义的函数

解析：
按照以上 AO 执行顺序，先处理参数得 x = 100，再处理函数声明，发现命名冲突，结论可看出，如果函数与参数命名冲突，参数被覆盖

解析：
按照以上 AO 执行顺序，先处理参数得 x = 100，再处理函数声明，接着声明了变量 fn，命名与函数 fn 重名，结论可看出，如果函数声明与变量声明重名，变量声明会被忽略

解析：
按照以上 AO 执行顺序，先处理参数得 x = 100，再处理函数声明，接着声明了变量 fn 并赋值为 1，命名与函数 fn 重名，结论可看出，如果函数声明与变量声明重名，变量声明并赋值时，函数声明被赋盖

由此可知，声明前置，同名函数覆盖变量
